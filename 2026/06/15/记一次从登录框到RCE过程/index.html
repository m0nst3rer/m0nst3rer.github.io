总结

整体攻击链路如下：

  1. 从前端页面和 JS 审计中识别产品指纹、登录接口、找回密码接口、后台文件管理和在线编辑接口。
  2. 在 retrieve.html 中发现 找回密码逻辑缺陷：前端校验邮箱验证码，但后端真正执行改密的接口没有校验验证码。
  3. 利用该逻辑缺陷直接重置管理员密码。
  4. 调用 /verify-user 登录接口，拿到管理员 JSESSIONID。
  5. 在后台 file.html 中发现 Web 根目录直接暴露为 D:/webapps/ROOT。
  6. 在 web_xml.html 中发现 dev/ide/save 可按 JSON 结构直接保存远程文件内容。
  7. 利用 sys/tool/file/add-file 先创建远程 JSP，再利用 dev/ide/save 把 JSP 内容写入 Web 根目录，实现 RCE。
